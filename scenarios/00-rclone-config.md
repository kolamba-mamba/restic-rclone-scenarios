# Настройка Rclone

Как создать подключения для Google Drive и обычных папок.

---

## 1. Запуск настройки

```bash
rclone config
```

## 2. Настройка Google Drive (gdrive)

1. Выбор `n` (New remote).
2. Указание имени: `gdrive`.
3. Выбор типа хранилища: `drive` (Google Drive).
4. **Client ID** и **Client Secret**: нажатие Enter для пропуска.
5. **Scope** (доступ):
   * `1` — полный доступ.
   * `3` — доступ только к файлам, созданным rclone.
6. **Service Account File**: нажатие Enter.
7. **Edit advanced config?**: `n` (No).
8. **Use auto config?**: `y` (Yes). Открытие браузера для входа в аккаунт.
9. **Configure as a Shared Drive?**: `n` (No).
10. Подтверждение: `y` (Yes, this is OK).

---

## 3. Настройка на сервере без браузера (SSH)

Если на сервере нет графического интерфейса, используются следующие способы:

### Способ А: Проброс порта через SSH

Перенаправление порта с сервера на свой компьютер:

1. Подключение по SSH с параметром `-L`:
   ```bash
   # Если порт SSH стандартный (22):
   ssh -L 53682:127.0.0.1:53682 user@remote-server-ip

   # Если порт другой (например, 2222):
   ssh -p 2222 -L 53682:127.0.0.1:53682 user@remote-server-ip
   ```
2. Выбор на сервере **Use auto config? `y`**.
3. Открытие в браузере на своем компьютере ссылки, которую выдает сервер.
4. Завершение входа.

### Способ Б: Передача токена вручную

1. Выбор `n` (No) на сервере в пункте **Use auto config?**.
2. Запуск команды `rclone authorize "drive"` на своем компьютере.
3. Вход в аккаунт в браузере и копирование полученного кода (JSON) в консоль сервера.

## 4. Настройка локальной папки (local_drive)

Предназначена для удобного обращения к папкам в скриптах.

1. Выбор `n` (New remote).
2. Указание имени: `local_drive`.
3. Выбор типа хранилища: `local` (Local Disk).
4. В пункте `Specify the root directory`:
   * **Пусто**: в скриптах указывается полный путь (например, `local_drive:D:/Backups`).
   * **Путь к папке**: `local_drive:` всегда будет указывать на эту папку.
5. Подтверждение: `y`.

---

## 5. Настройка шифрования (crypt)

Создание защищенной папки поверх существующего подключения. Файлы шифруются до отправки в облако.

1. Запуск `rclone config`.
2. Выбор `n` (New remote).
3. Указание имени: `secret_drive`.
4. Выбор типа: `crypt` (Encrypt/Decrypt a remote).
5. **Remote to encrypt/decrypt**: путь к папке в облаке, где будут лежать зашифрованные файлы (например, `gdrive:encrypted_backup`).
6. **How to encrypt filename**: `1` (шифровать имена файлов).
7. **How to encrypt directory names**: `1` (шифровать имена папок).
8. **Password or script for encryption**: ввод пароля для файлов.
9. **Password or script for salt**: ввод второго пароля (соли).
10. Подтверждение: `y`.

Для работы в скриптах используется имя защищенного подключения (например, `secret_drive:` вместо `gdrive:backup`). Rclone выполняет шифрование при загрузке и расшифровку при чтении автоматически.

---

## 6. Как указывать пути

Настройка в `rclone config` — это только точка входа. Сами папки записываются следующим образом:

* **Корень:** `gdrive:`
* **Папка в корне:** `gdrive:backup_folder`
* **Вложенная папка:** `gdrive:Backups/Server1`

Если папки нет, программы создают её самостоятельно при первом бэкапе.

---

## 7. Проверка связи

Проверка правильности настройки:

### Шаг 0: Определение пути к настройкам

Для получения пути к файлу настроек для скриптов:

```bash
rclone config file
```

### Шаг 1: Проверка связи (Rclone)

Просмотр списка папок в хранилище:

```bash
rclone lsd gdrive:backup_folder
```

*Наличие папок data, index, keys подтверждает наличие связи.*

### Шаг 2: Проверка бэкапов (Restic)

Просмотр списка копий:

```bash
restic -r rclone:gdrive:backup_folder snapshots
```

*Если программа запрашивает пароль и выводит таблицу — система работает.*

---

## 8. Работа с Restic

Создание нового хранилища Restic через Rclone:

```bash
restic -r rclone:gdrive:backup_folder init
```

*Слово `gdrive` — это имя, которое вы дали при настройке.*
