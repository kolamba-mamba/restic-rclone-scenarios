# Управление данными

Просмотр списка копий, восстановление файлов и удаление устаревших данных.

---

## 1. Список копий (Snapshots)

Для получения информации о существующих бэкапах используются следующие команды:

### Общий список:
```bash
restic -r $RESTIC_REPOSITORY snapshots
```

### Фильтрация по меткам или путям:
```bash
# Поиск копий с меткой "local"
restic -r $RESTIC_REPOSITORY snapshots --tag "local"

# Поиск копий для конкретной директории
restic -r $RESTIC_REPOSITORY snapshots --path "/home/user/data"
```
*Каждая копия идентифицируется уникальным ID (например, `a1b2c3d4`), который используется для операций с конкретным бэкапом.*

---

## 2. Очистка хранилища

Удаление ненужных копий позволяет освободить место.

### Автоматическая ротация:
```bash
# Сохранение только 10 последних копий
restic -r $RESTIC_REPOSITORY forget --keep-last 10
```

### Удаление по ID:
```bash
# Удаление одной или нескольких копий
restic -r $RESTIC_REPOSITORY forget snapshot_id1 snapshot_id2
```

### Физическое удаление данных (Prune):
Команда `forget` только исключает копии из индекса. Для физического удаления данных и освобождения места запускается очистка:
```bash
# Полная очистка хранилища
restic -r $RESTIC_REPOSITORY prune

# Удаление копии с немедленной очисткой места
restic -r $RESTIC_REPOSITORY forget snapshot_id --prune
```

---

## 3. Восстановление файлов (Restore)

Процесс извлечения данных из хранилища в целевую папку.

### Восстановление последней копии:
```bash
restic -r $RESTIC_REPOSITORY restore latest --target /path/to/restore
```

### Восстановление по ID:
```bash
restic -r $RESTIC_REPOSITORY restore snapshot_id --target /path/to/restore
```

### Выборочное восстановление:
Путь в `--include` указывается в том виде, в котором он хранится внутри snapshot. 
```bash
# Извлечение конкретного файла из бэкапа
restic -r $RESTIC_REPOSITORY restore latest --target /tmp/restore --include "/home/user/important.docx"
```

### Просмотр путей внутри копии:   
Перед выборочным восстановлением используется просмотр содержимого snapshot:
```bash
restic -r $RESTIC_REPOSITORY ls latest
```

### Особенности путей в Linux и Windows:
*   **Linux:** При хранении полных путей восстановление в корень (`--target /`) возвращает данные в привычную структуру системы.
*   **Windows:** При хранении полных путей восстановление в корень диска создает структуру вида `C:\C\...`. Параметр `--include` не меняет это поведение и служит только для выборочного восстановления нужного файла или каталога.

---

## 4. Подключение как диска (Mount)

Хранилище подключается как обычная директория для работы через проводник или файловый менеджер.

### В Windows (требуется [WinFSP](https://winfsp.dev/)):
```powershell
# Подключение хранилища как диска M:
restic -r $env:RESTIC_REPOSITORY mount M:
```

### В Linux (требуется [FUSE](https://github.com/libfuse/libfuse)):
```bash
mkdir /mnt/restic
# Подключение хранилища в папку /mnt/restic
restic -r $RESTIC_REPOSITORY mount /mnt/restic
```

---

## 5. Проверка целостности (Check)

Для подтверждения исправности хранилища применяются различные режимы проверки:

*   **Структурная проверка (`check`):** анализ целостности индекса и наличия всех блоков данных.
*   **Полная проверка (`check --read-data`):** перечитывание всех данных из хранилища (наиболее надежный метод).
*   **Выборочная проверка (`check --read-data-subset=10%`):** чтение 10% случайно выбранных данных.
